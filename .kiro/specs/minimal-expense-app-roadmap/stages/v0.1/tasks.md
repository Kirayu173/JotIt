# Implementation Plan

- [ ] 1. 初始化移动端项目骨架与基础导航
  - [ ] 1.1 创建 `React Native + Expo + TypeScript` 项目骨架，并接入 `记账 / 回顾 / 设置` 一级导航
    - 配置基础目录结构，至少包含 `app`、`features`、`domain`、`data`、`components`、`hooks`
    - 建立浅色优先的基础主题、颜色变量和页面容器
    - 预留 Android 优先的运行配置，不引入账号、云同步、AI 或 OCR 依赖
    - _Requirements: 1.1, 1.6, 1.8, 6.2, 6.3_
  - [ ] 1.2 建立全局状态与应用初始化流程
    - 接入 `Zustand + Repository Hooks` 的状态组织方式
    - 初始化 SQLite 连接、应用设置加载和默认币种 `CNY`
    - 为后续功能提供统一的 app shell 和错误边界
    - _Requirements: 1.2, 1.7, 6.1, 6.3_

- [ ] 2. 实现 v0.1 核心数据模型与数据库结构
  - [ ] 2.1 定义交易、分类、总结快照和应用设置模型
    - 用 TypeScript 定义 `Transaction`、`Category`、`SummarySnapshot`、`AppSettings`
    - 约束 `v0.1` 仅支持人民币，统一使用 `CNY`
    - 为软删除、撤销和时间范围查询预留必要字段
    - _Requirements: 1.2, 1.4, 1.5, 1.7, 6.1_
  - [ ] 2.2 创建 SQLite schema、迁移和 repository 接口
    - 建立 `transactions`、`categories`、`summary_snapshots`、`app_settings` 表
    - 编写交易新增、更新、软删除、按区间查询、按天分组查询接口
    - 编写单元测试验证本地持久化、软删除和时间范围查询行为
    - _Requirements: 1.2, 1.3, 1.4, 6.1, 6.3_

- [ ] 3. 实现默认分类与基础设置初始化
  - [ ] 3.1 预置默认分类并支持自定义分类写入
    - 写入 8 个支出分类：餐饮、交通、日用、购物、娱乐、居住、医疗、其他
    - 写入 3 个收入分类：工资、报销、其他
    - 支持后续在设置页新增自定义分类，但不实现二级分类
    - _Requirements: 1.1, 1.2, 1.3, 6.1_
  - [ ] 3.2 初始化 v0.1 基础设置
    - 保存默认币种、默认回顾粒度为“月”、记账列表加载窗口为 365 天
    - 保存列表分组方式为“按天分组”
    - 为后续设置页读取提供统一配置接口
    - _Requirements: 1.5, 1.7, 6.1_

- [ ] 4. 实现记账页列表与上拉加载体验
  - [ ] 4.1 构建记账页轻量概览与按天分组的记录列表
    - 展示今日支出、本月支出、本月结余的轻量概览
    - 按时间倒序、按天分组渲染记录，单条记录展示分类、备注、金额和时间
    - 只加载最近 365 天数据，支持向上滑动分页加载
    - _Requirements: 1.1, 1.3, 1.7_
  - [ ] 4.2 实现记账页的列表查询与同步刷新
    - 新增、编辑、删除后自动刷新列表和轻量概览
    - 对空列表、加载失败、超出时间门限等状态提供清晰提示
    - 编写集成测试验证记账页刷新与按天分组行为
    - _Requirements: 1.2, 1.3, 1.4, 6.3_

- [ ] 5. 实现底部大弹窗记账流程
  - [ ] 5.1 构建从底部拉起的记账弹窗 UI
    - 在记账页放置悬浮主按钮，点击后拉起覆盖大约 65%-75% 屏幕的底部弹窗
    - 在弹窗中提供收入/支出切换、大字号金额区、分类区、备注区、日期时间区
    - 金额输入使用系统数字键盘，不引入自定义计算器键盘
    - _Requirements: 1.1, 1.3, 1.7_
  - [ ] 5.2 实现交易创建与连续记账交互
    - 默认使用当前日期时间，并允许用户手动修改
    - 支持“保存”和“保存并继续”两种操作
    - 保存成功后将交易持久化到本地，并同步更新记账页和回顾页数据
    - 编写单元测试与集成测试验证创建流程和字段校验
    - _Requirements: 1.1, 1.2, 1.7, 6.3_

- [ ] 6. 实现记录编辑、软删除与短时撤销
  - [ ] 6.1 构建记录编辑入口与编辑表单复用
    - 点击记账页或回顾页中的记录可进入编辑态
    - 复用记账弹窗的数据结构和表单组件，避免维护两套录入逻辑
    - 保存编辑后同步更新统计和列表
    - _Requirements: 1.3, 1.4, 6.1_
  - [ ] 6.2 实现软删除与撤销条交互
    - 删除记录时写入软删除标记，而不是直接永久移除
    - 删除后展示短时撤销条，支持快速恢复
    - 编写测试验证删除、撤销和统计一致性
    - _Requirements: 1.4, 6.1, 6.3_

- [ ] 7. 实现回顾页时间范围查询、统计和同页流水
  - [ ] 7.1 构建回顾页的时间粒度与区间切换能力
    - 默认打开“月”视图
    - 支持 `周 / 月 / 季 / 年 / 自定义区间` 切换
    - 将任意时间粒度统一映射到同一套查询与聚合接口
    - _Requirements: 1.5, 6.1, 6.5_
  - [ ] 7.2 实现回顾页概览卡片、图表占位和同页流水
    - 展示总收入、总支出、净结余、主要支出分类
    - 实现一类基础图表，推荐优先做分类占比
    - 在同页展示当前时间范围下的流水列表，并支持点击编辑
    - _Requirements: 1.3, 1.4, 1.5_

- [ ] 8. 实现规则型自动总结与月末快照生成
  - [ ] 8.1 编写规则型总结引擎
    - 基于时间范围内的交易生成总收入、总支出、净结余、最大支出分类、环比变化和高频分类提示
    - 输出结构化总结对象和简短自然语言文案
    - 编写单元测试验证不同时间粒度和空数据场景
    - _Requirements: 1.5, 6.1, 6.5_
  - [ ] 8.2 生成月末总结快照并接入回顾页
    - 在月份结束或切换月份时生成当月总结快照
    - 支持手动查看任意月份和任意区间总结
    - 当规则结论更新时，同步刷新回顾页中的总结卡片
    - _Requirements: 1.5, 6.1, 6.3_

- [ ] 9. 实现 v0.1 设置页与基础质量保障
  - [ ] 9.1 构建设置页的最小功能集
    - 提供分类管理入口、自定义分类创建、数据说明和关于页
    - 明确展示“本地存储、离线可用、无账号依赖”的产品状态
    - 不实现导出、备份、AI、OCR、云同步入口
    - _Requirements: 1.6, 1.8, 6.3, 6.4_
  - [ ] 9.2 补齐 v0.1 的自动化测试与完成标准
    - 为 repository、记账弹窗、记账页列表、回顾页聚合和规则总结引擎补齐自动化测试
    - 验证核心路径：新增、编辑、删除、撤销、按区间回顾、月末总结
    - 确保项目在无网络、无 AI 配置下可独立运行
    - _Requirements: 1.2, 1.4, 1.5, 6.2, 6.3, 6.4_
