# Implementation Plan

- [ ] 1. 扩展 v0.2 领域模型与数据库迁移
  - [ ] 1.1 定义周期交易、到期项、搜索筛选与备份载荷模型
    - 新增 `RecurringRule`、`RecurringOccurrence`、`LedgerSearchFilters`、`BackupPayloadV2`
    - 复用现有 `TransactionRecord`、`Category`、`SummarySnapshot`、`AppSettings`
    - 明确 `v0.2` 仍然只支持 `CNY`
    - _Requirements: 2.1, 2.2, 2.4, 2.5, 6.1_
  - [ ] 1.2 实现数据库迁移到 `user_version = 2`
    - 新建 `recurring_rules`、`recurring_occurrences` 表
    - 为交易搜索和恢复补充必要索引
    - 更新 repository 接口与基础测试
    - _Requirements: 2.1, 2.3, 2.5, 6.1, 6.3_

- [ ] 2. 实现周期交易规则与到期项生成能力
  - [ ] 2.1 编写周期规则 repository 与服务层
    - 支持新增、编辑、暂停、启用、删除规则
    - 规则频率先支持 `周 / 月 / 年`
    - 计算下一个到期日并避免重复生成
    - _Requirements: 2.1, 2.6, 6.1_
  - [ ] 2.2 实现到期项确认 / 跳过流程
    - 到期后生成 `pending` occurrence
    - 用户确认后写入真实交易，`source = recurring`
    - 用户跳过后记录状态并同步刷新列表与统计
    - _Requirements: 2.1, 2.6, 6.1, 6.3_

- [ ] 3. 实现记账页历史搜索与筛选
  - [ ] 3.1 扩展交易查询接口
    - 支持按月份、分类、收支类型、关键词筛选
    - 尽量在 SQLite 层完成查询与排序
    - 保持结果仍可按天分组渲染
    - _Requirements: 2.3, 2.6, 6.1_
  - [ ] 3.2 构建记账页搜索与筛选交互
    - 增加顶部搜索框与筛选入口
    - 支持清空筛选、空结果状态、错误状态
    - 搜索 / 筛选后仍支持编辑、删除、撤销
    - _Requirements: 2.3, 2.6, 6.3_

- [ ] 4. 实现轻量录入候选，不做快捷模板管理
  - [ ] 4.1 统计近期常用分类与常用金额
    - 基于最近真实交易计算高频分类
    - 提供少量常用金额候选
    - 不新增独立模板管理表或模板编辑页
    - _Requirements: 2.2, 6.1_
  - [ ] 4.2 在记账弹窗中接入候选提示
    - 新建记录时显示常用分类 / 常用金额候选
    - 周期交易待确认进入弹窗时自动带入草稿值
    - 保持弹窗主流程仍然是“金额 -> 分类 -> 保存”
    - _Requirements: 2.1, 2.2, 6.3_

- [ ] 5. 实现 `CSV` 导出与 `JSON` 备份
  - [ ] 5.1 编写导出服务与数据序列化
    - `CSV` 导出交易明细，包含分类名与基础元信息
    - `JSON` 导出完整备份载荷，包含 schema 版本
    - 明确导出文件不会被隐式上传
    - _Requirements: 2.4, 2.5, 6.3, 6.4_
  - [ ] 5.2 接入设置页的数据管理入口
    - 在设置页新增 `CSV` 导出、`JSON` 导出按钮
    - 支持本地保存或分享
    - 对取消、失败、成功给出清晰反馈
    - _Requirements: 2.4, 2.6, 6.3_

- [ ] 6. 实现 `JSON` 恢复与恢复前保护机制
  - [ ] 6.1 编写恢复校验与事务性写入流程
    - 校验 `schemaVersion`、关键字段、主键关系
    - 恢复前生成当前数据库临时快照
    - 用事务执行清空并重建本地数据
    - _Requirements: 2.5, 2.6, 6.1, 6.3_
  - [ ] 6.2 实现恢复 UI 与结果反馈
    - 在设置页增加恢复入口与二次确认
    - 恢复完成后刷新全局状态与回顾快照
    - 恢复失败时保留原数据并展示失败原因
    - _Requirements: 2.5, 2.6, 6.3_

- [ ] 7. 补齐 v0.2 自动化测试与验收
  - [ ] 7.1 编写单元与集成测试
    - 验证周期交易生成、确认、跳过逻辑
    - 验证搜索筛选结果正确性
    - 验证 `CSV` 导出与 `JSON` 备份结构
    - 验证导出后恢复的数据 round-trip 一致性
    - _Requirements: 2.1, 2.3, 2.4, 2.5, 2.6, 6.3_
  - [ ] 7.2 完成 v0.2 阶段验收清单
    - 确认无 AI 配置时功能仍完整可用
    - 确认恢复前保护机制有效
    - 确认“常用候选已接入，但未引入快捷模板管理”
    - _Requirements: 2.2, 2.5, 6.2, 6.3, 6.4_
